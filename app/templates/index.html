Why coviolations?

With coviolations you can:

  • checking your code with pep8, coverage and many other tools;
  • create fancy commit representation;
  • control code quality with trending graphs;
  • show fancy test result representation.

How it works?

When you push code to github repository, your ci tool run violations and tests on your code and send to coviolations. After that, coviolations parse test and violations result and create fancy representation.

How-to use with

  1. Create .covio.yml with violations and project name: 
    violations:
  pep8: pep8 . --exclude='*migrations*'
  sloccount: sloccount .
  2. Enable project in management page.
  3. Add dependency installation and covio call to .travis.yml: 
    before_install:
  - pip install pep8
  - sudo apt-get update -qq
  - sudo apt-get install -qq sloccount
after_script:
  - pip install coviolations_app
  - covio
